BountyOps Claim Guardian

A reasoning agent that protects open bounty payouts by monitoring GitHub, Gmail notifications, claim state, CI, and public evidence trails, then acting only when a real payout risk appears.

Agent Loop

  • Read watcher, claim, PR, issue, and inbox state.
  • Rank active lanes by payout value, merge readiness, and visibility risk.
  • Verify whether the latest public issue tail still preserves the active claim.
  • Post a concise public update only when evidence shows a real gap.
  • Write compact checkpoints so the next run resumes from current evidence.

Public Evidence

The demo uses public GitHub evidence and sanitized local fixtures. It excludes private mailbox bodies, live claim URLs, cookies, tokens, payout credentials, banking data, and authenticated claim pages.
